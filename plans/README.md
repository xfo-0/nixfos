## Findings considered and rejected (don't re-audit)
- **`includes = lib.attrValues тАж._` "circular" (persist/minimal.nix)** тАФ not a
  bug; this is the documented den idiom for pulling all `_`/`provides` children.
- **`.envrc` has `use nix` commented out** тАФ correct: there is no devShell, and
  dev tools (sops, ssh-to-age, nh, moor, nxopt) are provided via home-manager, so
  there's nothing to `use`. Not a finding.
- **deploy.nu temp-dir "race"** тАФ `mktemp -d` creates the dir mode `0700`, so the
  decrypted host key is never world-readable. Only a missing cleanup `trap` on
  early error remains (cosmetic) тАФ see Notes.
- **sops template files "persisted plaintext" (git-credentials, gh-hosts.yml)** тАФ
  sops-nix renders templates to `/run` (tmpfs), not persisted; the symlinks point
  there. By design.
- **curl TLS not explicitly pinned** тАФ curl verifies via the system CA bundle by
  default; no action.

### Delta audit 2026-06-20 (changes `92ac7e5..6fa914f`) тАФ considered and rejected
- **nl0x home-assistant / vaultwarden "data loss under impermanence"** тАФ both are
  `enable = false` in `nl0x/configuration.nix` and each aspect gates config with
  `lib.mkIf cfg.enable`, so they're inert. Inclusion in the host's `includes` is
  declaration, not activation. **Not a finding *today*.** Caveat for the
  enable-path: `vaultwarden.nix` defines a `persist` block, but
  **`home-assistant.nix` does NOT** тАФ enabling HA on a prod (impermanence) host
  would wipe its state dir each boot. Add a persist block before enabling HA (see
  the nl0x bring-up note).
- **nl0x atuin-server "unpersisted"** тАФ `atuin-server.nix` has a `persist` block
  for `/var/lib/postgresql`; it fires when enabled. Persisted. Not a finding.
- **`run-repo.nu` / `smoke.sh` shell/SQL/nix-expr interpolation** (`^sh -c $cmd`,
  `select тАж where name='$query'`, `import ./ci/tests/$name.nix`) тАФ inputs are the
  user's own CLI args and moor-managed paths against a regenerable local cache,
  read-only. LOW, local personal tooling; hardening it would out-cost the risk.
- **`codebase-memory-mcp` prebuilt binary** тАФ third-party GH-release binary wired
  as an MCP server, but **hash-pinned** (`fetchurl` sha256) and x86_64-only; a
